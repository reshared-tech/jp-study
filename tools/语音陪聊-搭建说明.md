# 语音陪聊搭建说明（Mac 本机 · Siri ↔ 我们这套系统）

让 Siri 接的不是通用 Claude，而是 **jp-chat 系统的大脑**：每次请求都实时读
`ledger.json` + `target-n2.json` + `学习哲学.md`，注入今日计划、到期项、已学词表、
最高规则，再调 Claude API。聊完追加到 `日志/voice-transcript-YYYY-MM-DD.md`，
之后回 `/jp-chat` 回写账本。

**全程在 Mac 本机闭环**，不用 Tailscale、不用 iPhone、不用公网。地址就是 `127.0.0.1`。

```
Mac 听写(日语) → POST 到 127.0.0.1:8765/jp → 脚本拼系统提示词+调API → {speak,text}
                                            → 朗读 speak(纯日语) + 显示 text(中文讲解)
```

---

## 1. 启动服务（Mac 终端，前台跑）

支持 **Claude / Gemini 可切**，环境变量 `JP_PROVIDER` 选,默认 claude。

**用 Gemini Flash（省钱快）：**
```bash
cd ~/Documents/jp-study
JP_PROVIDER=gemini GEMINI_API_KEY=AIza... python3 tools/jp_voice_server.py
```

**用 Claude（准、守格式）：**
```bash
cd ~/Documents/jp-study
ANTHROPIC_API_KEY=sk-ant-xxx python3 tools/jp_voice_server.py
```

启动后会打印 `供应商：gemini  模型：gemini-3.5-flash`。自测：
```bash
curl -s http://127.0.0.1:8765/health
# {"ok":true,...,"provider":"gemini","model":"...","has_key":true}
curl -s -X POST http://127.0.0.1:8765/jp -H 'content-type: application/json' \
     -d '{"text":"今日は何を勉強する？"}'   # 应返回 {"speak":...,"text":...}
```

**可调的环境变量：**
- `JP_PROVIDER` = `gemini` / `claude`
- `JP_MODEL` = 覆盖模型 ID（Gemini ID 拿不准时改这个，例如换成 `gemini-2.5-flash`，不用动代码）
- `JP_MAX_TOKENS` = 单次最多输出 token（默认 1024）
- `JP_VOICE_PORT` = 端口（默认 8765）

> ⚠️ 若 Gemini 报 `api 404 / model not found`，多半是模型 ID 不对——去 `?key=` 那个
> ListModels 接口查准确 ID，再用 `JP_MODEL=` 覆盖。
> 若返回空、`text` 为空：把 `JP_MAX_TOKENS` 调大（flash 思考会吃 token）。

（想后台常驻、开机自启，之后可做 launchd plist；先手动跑通。）

## 2. Mac 快捷指令（6 步）

打开 macOS 自带的「快捷指令」App，新建一个，命名「练日语」：

1. **听写文本**（Dictate Text）— 语言：日语。
2. **获取 URL 内容**（Get Contents of URL）
   - URL：`http://127.0.0.1:8765/jp`
   - 方法：`POST`
   - 请求头：`Content-Type` = `application/json`
   - 请求体：`JSON` → 加一个字段 `text`，值选第 1 步的「听写文本」。
3. **获取词典值**（Get Dictionary Value）— 键 `speak`（从第 2 步结果取）。
4. **朗读文本**（Speak Text）— 内容 = 第 3 步；语音选日语（Kyoko / O-ren）。
5. **获取词典值**（Get Dictionary Value）— 键 `text`（从第 2 步结果取）。
6. **显示结果 / 快速查看**（Show Result / Quick Look）— 内容 = 第 5 步（看汉字+假名+讲解）。

可在快捷指令的「i」设置里勾选「用 Siri 运行」，之后喊「嘿 Siri，练日语」即可；
也可以直接在快捷指令 App 里点运行，或拖到菜单栏/程序坞一键启动。

## 用法
- 运行「练日语」→ 说一句（中日混说都行）→ 听它读日语示范 + 看屏幕讲解。
- 想换话题：说 **「重置」/「新话题」**，清空多轮历史重新开。
- 多轮对话历史存在 `tools/.voice_session.json`（已 gitignore），自动续上。

## 回写账本（慢道）
聊一段后，回到 Claude Code 跑 `/jp-chat`，告诉我「读今天的 voice-transcript」，
我就按里面的内容更新 ledger / target-n2 / 日志 / 跟读音频——账本不丢。

## 排错
- 浏览器开 `http://127.0.0.1:8765/health` 应返回 ok；不行就是服务没启动。
- 返回 500 `no ANTHROPIC_API_KEY`：启动时没带 key。
- 读出来是一坨 JSON：第 3 步键名要填 `speak`（不是整个结果）。
- 中文被日语音色读得怪：正常，朗读只接 `speak`（纯日语），中文只在屏幕显示。
- 第一次用听写/朗读：系统设置 → 键盘 → 听写 打开；朗读音色在 系统设置 → 辅助功能 → 朗读内容 里加日语音色。
